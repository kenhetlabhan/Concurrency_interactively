About Lock-based Concurrency:

Lock-based Concurrent Data Structures Adding locks to a data structure makes the structure thread-safe, therefore the data structure is usable by threads and leads to a high-yielding performance. There are several different structures: Counters, Linked-lists, Queues, and Hash Tables.

There are 2 different types of counters:

Concurrent counter:

Counter: A device or function that stores the number of times equal values are added or events have occurred

Counter with locks: When a routine is called, a lock is acquired, and it is released when the call is returned.

Perfect scaling: Threads complete work on processors in parallel.

Approximate Counter: Helps counter the issue of bugs in counter scalability on multicore machines.

A single logical counter using multiple local physical counters, one per CPU core, and a single global counter.

A lock per local physical counter and the global counter.

Concurrent linked lists: Linked lists: An ordered structure which consists of nodes which have an element and a pointer

The issue with concurrent linked lists: Failure of function causes the lock to be released and must undo all the states before returning, causing bugs. Only surrounding locks and releases around the critical code area.

Scaling linked lists: Hand over-hand locking (Lock coupling)

Concurrent queues: Queue: First in first out system which orders a sequence of operations. Two locks to enable concurrency of enqueue and dequeue operations.
